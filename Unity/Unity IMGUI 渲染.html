# Unity IMGUI 渲染

# 扩展编辑器中怎么渲染UI的？

unity执行着一个循环程序，在循环执行的过程中有一个事件队列，在每一个事件中执行相关UI的渲染，比如，当调用GUILayout渲染UI时，循环程序首先会在EventType.Layout事件中计算控件渲染所需要的空间，，然后在EventType.Repaint方法中实际渲染控件

# 可能存在的问题

  • "ArgumentException: Getting control 0's position in a group with only 0 controls when doing Repaint." 类似的错误

当执行一个操作会导致接下来的控件渲染发生变化，例如控件根据一个列表渲染，修改了一个列表，由于之前已经根据控件顺序通过Layout事件计算了渲染控件需要的空间，这个修改导致了控件的顺序发生了变化，之后在Repaint事件中根据新的控件顺序渲染的时候就会报错

解决办法：

  1. 调用GUIUtility.ExitGUI方法阻止后续控件的渲染，直到下一次循环开始
  2. 在Repaint事件中修改数据
